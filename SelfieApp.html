The selfie app allows users to take a photo from the video stream and edit the photo using a drawing function, frames and filters. Once the user is done editing their photo they can upload it to a server.

The application reads in a video stream from a video source which can be a laptop camera or a web camera. Once the video stream is connected, JavaScript functions get used to interact with the stream on button clicks. The snapshot function takes a still image from the video stream and stores that image in the ‘source’ canvas and the pixel canvas to be called upon later. To make a frame appear, click either frame button, which are both linked with a click event listener. The draw function sees which frame has been selected and the frame is pulled from the asset manager cache and places the frame with the captured shot of the video stream into a canvas.

Similar to frames, an event listener is tied to the filter buttons which when clicked will change the image to either invert or greyscale. The buttons for drawing in the canvas are put in an unordered list. Clicking on the draw or erase button will call a simple function that sets the stroke size and the strokes colour. When erase all is clicked, a clean source image over the altered image, erasing any drawings, filters or frames. The save button opens a new window and loads the data in the canvas into the new window as a png image.

In the creation of this application, I learned how to use JavaScript and the HTML canvas to interact with a video stream from a camera. When the video stream is loaded into the canvas a button can be clicked to take a still image into another canvas which can be edited using filters and frames. A draw tool was created to allow users to draw, erase and change the colour and size of the drawing tool. I then created functions that allow the user to save their image to a new web browser or upload it to a server.
